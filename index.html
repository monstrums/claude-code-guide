Phase 1 — Foundations

Days 1–3

Day 1
The Mental Model: Claude Code is Not a Chatbot

Claude Code is an autonomous agent that lives in your terminal (the text-based app on your computer — like a command line). It can read files, write files, run commands, search the web, and connect to apps. You give it a task; it figures out which tools to use.

Which Claude am I using?

The Claude desktop app gives you access to three different modes — and they're not the same thing:

ModeWhat it isBest for
Claude ChatThe classic AI conversation. Ask questions, get answers. No file access or tool use.Quick questions, brainstorming, writing help
Claude CodeAn autonomous agent in your terminal. Reads/writes files, runs commands, connects to apps. This is what this guide teaches.Automating real work across your computer and apps
CoworkA visual desktop mode with file access and some Claude Code features (MCP, skills). Friendlier interface but fewer power features.File tasks, document creation, visual workflows

This guide teaches Claude Code (terminal) because it unlocks the full toolkit — skills, hooks, subagents, parallel sessions, and one-shot mode. Everything in this guide works in the terminal. If you already use Cowork, think of Claude Code as the power-user upgrade: same AI, more control.

What to learn

  • Claude Code runs in a loop: read context → reason → use a tool → repeat
  • Every file it reads, every message you send consumes context tokens (Claude's short-term memory)
  • When memory fills up, Claude gets less effective — this constraint shapes everything
  • Use /compact to free up memory, /clear to start fresh between tasks

Practice — Set up your briefing project

Step 0 — Open your terminal. This is the text-based app where you'll type commands. You only need to find it once — after that, Claude Code lives here.

  • Mac: Press Cmd + Space, type Terminal, hit Enter. (Or find it in Applications → Utilities → Terminal)
  • Windows: Press Win + R, type cmd, hit Enter. (Or search "Command Prompt" or "PowerShell" in the Start menu)

You'll see a blinking cursor waiting for input — that's it, you're in. Every command below gets typed here and confirmed with Enter.

Step 1 — Install Claude Code. Paste this into your terminal and hit Enter:

curl -fsSL https://claude.ai/install.sh | bash
claude --login

On Windows, visit claude.ai/download instead to get the installer.

Step 2 — Create your project folder and start Claude:

mkdir -p ~/weekly-briefing
cd ~/weekly-briefing
claude

mkdir creates a folder. cd moves into it. claude launches Claude Code inside that folder.

Step 3 — Ask Claude to generate sample files. Paste this prompt:

"Create a folder called raw-notes/ with 8 sample files that simulate a messy work week for someone in tech. Include things like: meeting-notes-monday.txt, slack-highlights.txt, project-alpha-update.txt, random-ideas.txt, design-review-feedback.txt, competitor-research.txt, action-items-scattered.txt, and weekly-goals.txt. Fill each with realistic content."

Step 4 — Verify. Ask: "How many files did you create? List them all with a one-line description of each."

Day 2
CLAUDE.md: Your Project's Memory

CLAUDE.md is a plain text file Claude reads at the start of every session. It tells Claude your rules and preferences. Think of it as a briefing doc for a new colleague.

What to learn

  • Run /init and Claude auto-generates a CLAUDE.md by scanning your project
  • Keep it under 200 lines — too long and Claude ignores parts
  • Write rules in plain English — this isn't code, just instructions
  • Claude also saves things it learns automatically between sessions

Practice — Teach Claude about your briefing project

Step 1 — Open your terminal (see Day 1 if you need a reminder) and start Claude in your project:

cd ~/weekly-briefing
claude

Step 2 — Ask Claude to create your CLAUDE.md:

"Create a CLAUDE.md file for this project. This is a 'Weekly Briefing System' — it takes messy notes from raw-notes/ and turns them into a clean weekly briefing. My rules: always write in plain language (no jargon), use bullet points, bold the project names, keep summaries under 5 items, put the date at the top, and never delete my original notes."

Step 3 — Ask: "Show me what's in the CLAUDE.md."

Step 4 — Test. Quit (/exit), relaunch claude, ask: "What are my rules?" — Claude should know without reminding.

Day 3
Permission Modes & Keyboard Flow

Permission modes control how much autonomy Claude gets. Default = asks permission. Accept Edits = just does it. Plan Mode = thinks but never touches. Switch with Shift+Tab.

ModeBehaviorWhen to use
DefaultAsks before every actionLearning, sensitive tasks
Accept EditsMakes changes without askingSpeed, trust
Plan ModeReads and thinks, never touchesAdvice before committing

Practice — Organize your notes using all three modes

  1. Plan Mode: Shift+Tab → "Look at raw-notes/. How would you categorize them? Suggest a folder structure — don't move anything."
  2. Accept Edits: Shift+Tab → "Create organized/ with subfolders. Copy (don't move) files into the right places."
  3. Default to verify: Shift+Tab → "Show folder structure of organized/ and confirm nothing in raw-notes/ was deleted."

Day 4
The Explore-Plan-Act-Verify Loop

The #1 pattern. Break any task into 4 phases: Explore → Plan → Act → Verify. Results are dramatically better than one big vague prompt.

The golden rule: Always give Claude a way to verify its own work — have it list what changed, compare before/after, or read back what it wrote.

Practice — Generate your first weekly briefing

  1. Explore: Plan Mode → "Read organized/ files. What were the main topics? Decisions? What's open?"
  2. Plan: "Write a briefing outline: Key Highlights, Decisions, Open Items, Priorities. Don't write it yet."
  3. Act: Accept Edits → "Create the briefing as output/weekly-briefing-2026-04-08.md. Follow my CLAUDE.md rules."
  4. Verify: "Read back the briefing. Under 1 page? Project names bolded? Date at top? Follows all rules?"

Day 5
Skills: One-Command Workflows

Skills are saved recipes — plain text instructions Claude follows when you trigger a slash command. Write once, use forever. /briefing now does what took you 10 minutes on Day 4.

Practice — Turn Day 4 into a one-command skill

Ask Claude: "Create a skill called 'briefing' at .claude/skills/briefing/SKILL.md. It should: read organized/, identify highlights/decisions/open items/priorities, write a briefing to output/ with today's date, verify against CLAUDE.md rules. Include $ARGUMENTS so I can focus on a specific topic."

Test it: type /briefing. Then try /briefing design to focus on one area.

Day 6
Subagents: Get More Done Without Losing Your Thread

A subagent is a separate Claude that works in isolation, then hands back just the result. Like sending three assistants to each research one topic — your main Claude's memory stays clean.

Practice — Richer briefing with parallel subagents

  1. Create a custom subagent: "Create .claude/agents/briefing-section.md for producing concise briefing sections."
  2. Use it: "Use a subagent for meeting files → 'Key Meetings' section. Another for project updates → 'Project Status'. Combine into one briefing."
  3. Compare: "Compare this to the Day 4 briefing. Which is more detailed?"

Day 7
MCP: Connecting Claude to Your Apps

MCP lets Claude talk to Slack, Notion, Google Drive, and hundreds more. One command to connect. Then Claude can read and write to your apps during any session.

Practice — Add live data to your briefing

Connect an app. Open your terminal (not inside Claude — exit first with /exit if needed). Pick the app you use most and paste the matching line:

claude mcp add slack -- npx @anthropic-ai/mcp-server-slack
claude mcp add notion -- npx @anthropic-ai/mcp-server-notion
claude mcp add gdrive -- npx @anthropic-ai/mcp-server-gdrive

These commands run in your regular terminal, not inside Claude Code. They register a connector that Claude can use in future sessions.

Now start Claude again (cd ~/weekly-briefing && claude) and ask: "Update my /briefing skill to also pull highlights from [your app]. Add a 'Team Highlights' section."

Day 8
Hooks: "Always Do This" Rules

Hooks are automatic actions that always fire — no AI involved, 100% reliable. Skills are recipes you trigger. Hooks are things that always happen.

Practice — Auto-archiving for your briefings

Ask Claude: "Set up two hooks in .claude/settings.json. First: every time you create or edit a file in output/, copy it to archive/ with a timestamp. Second: show a Mac notification when you finish any task."

Run /briefing to test. Check archive/ for the timestamped copy.

Day 9
Running Multiple Claudes at Once

Run several Claude sessions in parallel — each on a different task. Two terminal tabs, two reports, same time.

Practice — Weekly + monthly reports simultaneously

First, open two terminal windows. On Mac: open Terminal, then press Cmd + N for a second window. On Windows: open two Command Prompt windows from the Start menu.

Window 1:

cd ~/weekly-briefing
claude -n weekly

Then ask: "Run my /briefing skill"

Window 2:

cd ~/weekly-briefing
claude -n monthly

Then ask: "Read all briefings in output/ and archive/. Write a monthly summary."

Two reports running at the same time — done in the time it takes to make one.

Day 10
Putting It All Together

One-shot mode — quick tasks without a full session

These commands run in your regular terminal — no need to start a full Claude session. Just paste and hit Enter:

claude -p "Count briefing files in ~/weekly-briefing/output/"
claude -p "Read the latest briefing and list just the open items"
cat notes.txt | claude -p "Summarize this in 3 bullet points"

The -p flag means "just answer this one thing and exit." The cat ... | claude -p pattern feeds a file's contents to Claude as input.
